# 기능 명세: 내 커리큘럼 — 경로 저장 (P3)

본 문서는 사용자가 두 키워드 사이 경로를 "내 커리큘럼"으로 저장·관리하는 기능의 명세를 정의한다.
[MVP_SCOPE.md](MVP_SCOPE.md) P3 항목에 해당한다.
경로 시각화는 [FEATURE_PATH_VISUALIZATION.md](FEATURE_PATH_VISUALIZATION.md), 학습 상태 마킹은 [FEATURE_NODE_STATUS.md](FEATURE_NODE_STATUS.md)를 참조한다.

## 1. 목적

사용자가 탐색한 두 키워드 사이의 학습 경로를 자신의 커리큘럼으로 보존하고,
이후 다시 불러와 시각화할 수 있게 한다.
"내가 어떤 학습 경로를 계획·검토했는가"를 누적·재탐색하는 통로다.

## 2. 동작 규약

### 2.1 저장 단위 — 경로 스냅샷
한 건의 커리큘럼은 다음 정보를 포함한다.
- 시작 키워드(A)와 종점 키워드(B)의 ID.
- **저장 시점의 경로상 중간 키워드 ID 목록** (스냅샷).
- 사용자가 입력한 **이름**.
- 사용자가 입력한 **메모(설명)** — 선택 입력.
- 생성 시각.

학습 상태 마킹([FEATURE_NODE_STATUS.md](FEATURE_NODE_STATUS.md))은 커리큘럼에 포함되지 않는다.
커리큘럼은 "탐색한 경로"만 기록하고, 키워드별 학습 상태는 별개 트랙으로 관리된다.

### 2.2 다중 저장 / 이름·메모
- **여러 개 저장 가능**. 개수 제한은 두지 않는다(시연용).
- 저장 시 사용자는 **이름**을 직접 입력한다.
- **메모(설명)** 필드도 함께 입력 가능 (선택).
- 이름 중복 허용 여부: 허용한다(시연 단순화).

### 2.3 저장 트리거 — 명시적 "저장" 버튼
- 두 키워드를 선택해 경로가 시각화된 상태에서 사용자가 **"저장" 버튼**을 누를 때만 저장된다.
- 자동 저장은 하지 않는다.
- 저장 버튼은 헤더 또는 그래프 영역 근처에 위치한다(정확한 위치는 후속 결정).
- 누르면 이름·메모 입력 모달이 뜨고, 확정 시 저장된다.

### 2.4 저장 위치 — 세션 메모리
- 저장 데이터는 **세션 메모리**에만 보관한다.
- 페이지 새로고침/탭 종료 시 초기화된다.
- `localStorage` 영속은 적용하지 않는다 ([MVP_SCOPE.md](MVP_SCOPE.md) §0의 mock 원칙 + 시연 단순화).
- 학습 상태 마킹과 동일한 휘발성 정책.

### 2.5 불러오기 UI — 사이드바 목록
- 좌측 사이드바에 **저장된 커리큘럼 목록**을 표시한다.
- 각 항목은 이름(+ 부가로 A→B 라벨, 메모 일부)을 보여준다.
- 항목 클릭 시 즉시 불러온다.

### 2.6 불러오기 동작 — 즉시 시각화
- 항목 클릭 시 헤더의 A·B 슬롯이 자동으로 채워지고, 경로가 즉시 시각화된다.
- 별도 확인 모달 없이 현재 상태를 덮어쓴다.
- 그래프가 정적 mock이므로 (저장 시점 ≠ 불러오기 시점)이라도 동일 경로가 산출된다.
  스냅샷된 중간 키워드 목록은 저장 당시의 기록으로만 보존되며, 본 시연 단계에서 시각화 결과에는 영향을 주지 않는다.

### 2.7 삭제 / 수정 — 삭제만 가능
- 각 항목에는 **삭제** 액션만 제공한다.
- 이름·메모의 수정은 지원하지 않는다.
- 삭제 시 별도 확인 모달은 두지 않는다(시연 단순화).
  필요 시 후속 작업에서 추가 가능.

## 3. 데이터 모델

```ts
type CurriculumEntry = {
  id: string;            // 내부 식별자 (uuid 또는 timestamp 기반)
  name: string;          // 사용자 입력 이름
  memo?: string;         // 사용자 입력 메모 (선택)
  path: {
    a: string;           // 시작 키워드 ID
    b: string;           // 종점 키워드 ID
    intermediates: string[]; // 저장 시점 경로상 중간 키워드 ID 목록
  };
  createdAt: string;     // ISO 문자열
};

// 세션 메모리 상태
type CurriculumStore = CurriculumEntry[];
```

## 4. 현재 프론트엔드와의 갭

| 항목 | 현재 ([App.jsx](../src/App.jsx)) | 본 명세 |
|---|---|---|
| 커리큘럼 데이터 모델 | 없음 | `CurriculumEntry[]` 세션 메모리 |
| 저장 버튼 | 없음 | 헤더/그래프 근처에 추가 |
| 이름·메모 입력 모달 | 없음 | 저장 시 표시 |
| 사이드바 목록 | 좌측에 학년 필터/토글만 존재 | 저장된 커리큘럼 목록 영역 추가 |
| 불러오기(클릭 → 헤더 채움 + 시각화) | 없음 | 항목 클릭 → 상태 갱신 |
| 삭제 액션 | 없음 | 항목별 삭제 버튼 |

## 5. 미결 사항
- 저장 버튼의 정확한 위치(헤더 우측 vs. 그래프 우상단 floating 등).
- 사이드바 레이아웃에서 학년 필터·"선택한 과목만 보기"와의 배치 우선순위.
- 사이드바 목록 항목의 표시 디자인 (이름·A→B·메모 미리보기 등).
- 이름 입력 모달의 폼 디자인.
