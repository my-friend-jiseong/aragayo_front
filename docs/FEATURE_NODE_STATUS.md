# 기능 명세: 학습 상태 마킹 (P2)

본 문서는 사용자가 그래프상 키워드 노드에 학습 상태를 마킹하고 시각적으로 구분하는 기능의 명세를 정의한다.
[MVP_SCOPE.md](MVP_SCOPE.md) P2 항목에 해당한다.
그래프 구조는 [GRAPH_ARCHITECTURE.md](GRAPH_ARCHITECTURE.md), 경로 시각화는 [FEATURE_PATH_VISUALIZATION.md](FEATURE_PATH_VISUALIZATION.md)를 따른다.

## 1. 목적

사용자가 학습 진척 상황을 그래프 위에서 직접 표시·확인할 수 있게 한다.
이미 학습한 개념과 앞으로 학습할 개념을 구분해 경로 탐색·학습 계획에 도움을 준다.

**중요한 역할 변화**: 본 명세 초안에서는 학습 상태가 시각 표시 전용이었으나,
[FEATURE_PATH_VISUALIZATION.md](FEATURE_PATH_VISUALIZATION.md) §2.2 변경에 따라
**"학습 완료" 상태가 경로 탐색의 능동적 입력(시작점)** 역할을 한다.
사용자가 어떤 키워드를 "학습 완료"로 마킹하느냐가 곧 학습 경로의 출발점 집합을 결정한다.

## 2. 동작 규약

### 2.1 상태 종류 — 3개
키워드 노드는 다음 중 하나의 상태를 가진다:
- **학습 완료**: 사용자가 학습을 마친 키워드.
- **학습 중**: 현재 학습 중인 키워드.
- **학습 예정**: 앞으로 학습할 의향이 있는 키워드.
- (그 외) **상태 없음 (기본)**: 마킹되지 않은 키워드.

마킹 대상은 키워드 노드뿐이며, Data Node에는 적용하지 않는다.

### 2.2 마킹 방법 — 노드 클릭 → 사이드 패널

- 노드를 클릭하면 화면 측면에 **상세 사이드 패널**이 열린다.
- 사이드 패널에서 상태(완료/중/예정/해제) 중 하나를 선택할 수 있다.
- 패널 외 영역 클릭 또는 닫기 버튼으로 닫는다.
- 한 번에 하나의 패널만 열린다.
- 노드 클릭은 헤더의 키워드 선택(드롭다운/검색)과 **별개의 동작**이다. 클릭으로는 A·B 슬롯이 변경되지 않는다.

### 2.3 시각 표현 — 색상 + 아이콘
상태별 시각 표현은 다음 두 채널로만 표현한다:

| 상태 | 색상 (예시) | 아이콘 (예시) |
|---|---|---|
| 학습 완료 | 초록 계열 | 체크(✓) |
| 학습 중 | 주황 계열 | 진행(↻) |
| 학습 예정 | 파랑/하늘 계열 | 별표(★) 또는 플래그 |
| 상태 없음 | 기본 회색 | 없음 |

**시각 채널 충돌 회피 원칙** ([FEATURE_PATH_VISUALIZATION.md](FEATURE_PATH_VISUALIZATION.md) §3.5 참조):
- 노드 크기 = GAT 가중치 (예약됨, 학습 상태와 무관).
- 후광/뱃지(halo) = GAT 추천도 높은 키워드 강조 (예약됨).
- 따라서 학습 상태는 **색상**과 **아이콘**만 사용해야 한다.
- 색상·아이콘은 정확한 디자인 토큰을 후속 결정한다 (위 표는 예시).

### 2.4 저장 위치 — 세션 메모리
- 상태 데이터는 **세션(메모리)**에만 저장한다. 페이지 새로고침 시 초기화.
- localStorage 영속 저장은 적용하지 않는다. (시연 단순화 목적.)

### 2.5 경로 시각화와의 상호작용 — 완료 노드 약화
경로 시각화 시점에 학습 상태에 따라 노드 표시가 변한다.

- **학습 완료** 키워드는 경로상에 있어도 **시각적으로 약화**한다.
  - 약화 방식: 채도/명도/투명도 감소.
  - 노드 자체는 경로에서 **제거하지 않는다** (경로 연속성 유지).
  - GAT 가중치 크기는 그대로 유지하되, 색·투명도가 죽어 사용자가 "이건 이미 안다"로 인지하도록.
- **학습 중**, **학습 예정**, **상태 없음** 키워드는 표시 약화 없이 정상 렌더링한다.

근거: 사용자가 P2-5에서 "(b)가 가능하면 (b), 아니면 (a)"라고 응답.
프론트 mock에서 (b)는 렌더링 단계 분기만으로 즉시 가능하므로 (b)로 채택. 단 경로 제거는 시각 흐름을 깨므로 약화에 한정.

### 2.6 필터 UI — 없음
- "완료만 보기 / 예정만 보기" 같은 필터 토글은 두지 않는다.
- 상태 구분은 색상·아이콘 같은 시각 채널로만 전달한다.

## 3. 현재 프론트엔드와의 갭

| 항목 | 현재 ([App.jsx](../src/App.jsx)) | 본 명세 |
|---|---|---|
| 학습 상태 데이터 모델 | 없음 | `{ keywordId: 'completed' \| 'in-progress' \| 'planned' }` 형태의 세션 상태 |
| 노드 클릭 핸들러 | 없음 | 클릭 시 사이드 패널 열기 |
| 사이드 패널 UI | 없음 (좌측 필터 사이드바만) | 우측 또는 별도 슬롯에 상세/상태 변경 패널 |
| 상태별 시각 채널 | 없음 | 색상 + 아이콘 차등 (크기·후광은 GAT 전용) |
| 경로 시 완료 노드 약화 | 없음 | 렌더링 단계에서 상태 조회 후 시각 효과 분기 |
| 영속 저장 | n/a | 세션 메모리만, localStorage 미사용 |

## 4. 미결 사항
- 색상 팔레트와 아이콘의 구체 디자인 토큰.
- 사이드 패널의 위치(좌/우)와 기존 좌측 학년 필터와의 레이아웃 관계.
- 사이드 패널에 표시할 부가 정보 범위 (예: 키워드 설명, 위키 링크, 관련 Data Node 미리보기 등 — 본 기능 외부 항목).
- "약화"의 정량 기준 (opacity 0.3? 0.4? 색 채도 N%?).
