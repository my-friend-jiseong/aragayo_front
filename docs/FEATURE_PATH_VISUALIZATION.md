# 기능 명세: 학습 완료 노드 → 목표 키워드 경로 시각화 (GAT 추천 통합)

본 문서는 헤더의 **목표 키워드 선택** 기능과, **학습 완료** 마킹된 키워드들로부터 목표까지의
경로를 다중 시작점 방식으로 시각화하는 동작에 대한 명세이다.
이 기능은 단순 경로 표시가 아니라 **GAT 추천 모델과 통합된 학습 경로 추천**이다.
그래프 구조 정의는 [GRAPH_ARCHITECTURE.md](GRAPH_ARCHITECTURE.md)를 따른다.

## 1. 목적

사용자가 학습 목표 키워드를 선택하면, **현재까지 "학습 완료"로 마킹된 모든 키워드에서
이 목표까지의 경로 중 GAT 추천 점수 상위 3개**를 시각화한다. 이를 통해 사용자는
자신의 학습 진척 상태를 기준으로 "앞으로 무엇을 더 학습해야 목표에 도달하는지"를 추천 받는다.

대표 사용 시나리오:
- 수강한 과목들을 "학습 완료"로 마킹한 뒤, 다음에 들을 과목을 목표로 지정해 갭을 확인.
- 학습 완료된 개념을 시작점으로 임의의 목표 개념까지의 학습 경로 확인.

## 2. 동작 규약

### 2.1 입력 — 목표 키워드 선택
- 헤더에 **단일 목표 키워드 슬롯**.
- **선택 대상은 그래프 내 모든 Keyword 노드**.
  - 위키피디아 기반 키워드, 과목 정보 기반 키워드 모두 포함.
  - 과목도 키워드의 한 종류이므로 별도 구분하지 않는다.
- 키워드 수가 많으므로 검색/자동완성 UI(`<datalist>`)로 입력한다.

### 2.2 시작점 — 학습 완료 키워드 집합
- 경로의 시작점은 **사용자가 입력하지 않는다**.
- 시작점은 [FEATURE_NODE_STATUS.md](FEATURE_NODE_STATUS.md)에서 "학습 완료"로 마킹된
  모든 키워드의 집합이다.
- 즉 본 기능은 사용자의 학습 상태를 능동적 입력으로 사용한다.

### 2.3 방향성 — 무방향 취급
- 키워드 간 간접 연결은 본질적으로 대칭이므로 시작점 → 목표 경로 탐색은 **무방향**으로 처리한다.
- (원본 간선은 Data Node → Keyword 단방향이지만, 키워드 간의 도달 가능성 자체는 양방향.)

### 2.4 경로 선택 — GAT 추천 점수 기준 Top 3
- 모든 시작점 → 목표 경로를 모은 뒤, **GAT 추천 점수가 높은 상위 3개 경로만** 강조한다.
  3개 미만이면 가능한 만큼만.
- 추천 점수 = 경로상 키워드들의 GAT 가중치 평균
  (`App.jsx`의 `gatPathScore`, 데이터 노드는 점수에 포함하지 않음).
- 동률 시 짧은 경로를 우선한다 (학습량이 적은 경로가 더 합리적이기 때문).
- 강조 개수는 `App.jsx`의 `TOP_PATHS`(=3) 상수로 관리한다.
- 탐색 단계의 안전 한도: 시작점당 `PATH_LIMIT(100)`, 합산 `PATH_LIMIT * 2(200)`.
- 최대 hop 수 기본값은 `4` (`pathSearch.js`의 `DEFAULT_MAX_HOPS`).

### 2.5 강조 대상 — 키워드만
- 경로상의 **Keyword 노드만** 강조한다.
- **Data Node는 시각화에 반드시 포함**한다. 단 강조 대상은 아니며 **흐림 처리**한다.
  - 숨기지 않는다. 데이터 노드는 [GRAPH_ARCHITECTURE.md](GRAPH_ARCHITECTURE.md) 정의상 그래프의 구조적 구성 요소이며,
    키워드 간 직접 간선은 존재하지 않으므로 데이터 노드를 가리면 키워드 간 연결이 거짓으로 표현된다.
  - 시각 표현: 작은 크기, 낮은 명도/채도, 라벨 미표시 (호버·확대 시 표시 가능).
  - Data Node에 대한 상세 인터랙션(사이드 패널·하이퍼링크 등)은 후속 기능에서 결정.
- 목표 키워드는 **빨간색 + 후광 + 큰 크기**로 단일 강조.
- 시작점(학습 완료 키워드)은 [FEATURE_NODE_STATUS.md](FEATURE_NODE_STATUS.md)의 초록 마킹을 그대로
  사용한다. 별도의 시작점 강조 색은 적용하지 않는다.
- 목표로 가는 경로에 기여하지 않는 학습 완료 키워드도 **동등하게 초록**으로 표시한다(별도 구분 없음).

### 2.6 예외 처리 — 경로 없음 / 시작점 없음
- 시작점(완료 키워드)이 하나도 없는 경우:
  **"키워드를 '학습 완료'로 마킹하면 목표까지 경로가 표시됩니다"** 안내 배너를 표시한다.
  목표 키워드 자체는 빨간색으로 강조한다. 그래프 자체는 정상 표시.
- 시작점은 있으나 목표로 가는 경로가 없는 경우:
  **"학습 완료된 노드 중 이 목표로 가는 경로가 없습니다"** 메시지를 표시한다.

## 3. GAT 추천 통합

본 기능은 GAT(Graph Attention Network) 추천 모델과 분리되지 않는다.
두 키워드 선택은 경로 시각화의 트리거이자 GAT 추천의 트리거이며,
추천 결과는 별도 패널·리스트 없이 **그래프 시각화에 직접 반영**된다.

### 3.1 추천 단위 — 키워드만
- GAT가 산출하는 우선순위는 **Keyword 노드 단위로만** 적용된다.
- Data Node는 본 단계에서 추천 단위가 아니다.

### 3.2 추천 트리거 — 목표 선택 + 학습 완료 마킹
- 추천은 **목표 키워드 입력** 또는 **학습 완료 상태 변경** 시 자동 재산출된다.
- 즉 학습 완료 노드 집합과 목표 키워드의 조합이 바뀔 때마다 경로가 갱신된다.

### 3.3 경로와 추천의 관계 — "경로 = 추천"
- GAT는 학습 완료 시작점들에서 목표 키워드까지의 **권장 학습 순서/우선순위**를 산출한다.
- 그 결과가 곧 경로 시각화의 내용이다. 별도의 "추천 리스트"는 생성되지 않는다.
- 점수 기준 상위 3개 경로(§2.4)만 그래프에 강조하고, 그 경로상 키워드의 GAT 가중치에 따라
  시각적 비중을 추가로 차등 적용한다.

### 3.4 추천 개수 — 상위 3개 경로
- 추천은 **점수 상위 3개 경로**로 제한한다. 더 많은 경로를 한꺼번에 보여주면 "추천"이라기보다
  "전체 도달 가능성 시각화"가 되어 사용자가 다음 단계를 고르기 어렵다.
- 경로 단위 Top 3이며, 키워드 단위 Top-N이 아니다(키워드 가중치 차등은 §3.5에서 별도).
- 임계 점수는 두지 않는다 — 항상 점수 상위부터 최대 3개.

### 3.5 그래프 위 시각 표현
경로상 키워드들의 GAT 가중치를 두 가지 시각 채널로 표현한다:
- **노드 크기 차등**: GAT 점수가 클수록 노드 반지름이 크다.
- **후광/뱃지**: 추천도가 특히 높은 핵심 키워드는 후광 또는 뱃지로 추가 강조한다.

별도의 사이드바·하단 패널·탭 영역은 사용하지 않는다.
모든 추천 정보는 **그래프 위 시각 표현으로만** 전달된다.

### 3.6 점수/이유 노출 — 비공개
- 수치 점수, 추천 이유 텍스트는 사용자에게 노출하지 않는다.
- 사용자는 노드 크기·후광 같은 시각적 위계로 우선순위를 인지한다.

## 4. 현재 프론트엔드와의 갭

| 항목 | 현재 ([App.jsx](../src/App.jsx)) | 본 명세 |
|---|---|---|
| 선택 대상 | `group === 'subject'`인 노드만 | 모든 Keyword |
| 선택 UI | `<select>` 드롭다운 2개 | 검색/자동완성 필요 |
| 강조 방식 | A, B 노드만 빨간색 + 후광 | 경로상 모든 키워드를 GAT 가중치 기반 크기·후광으로 차등 강조 |
| 경로 계산 | 없음 | 두 노드 사이의 모든 경로 탐색 로직 필요 |
| GAT 가중치 반영 | 없음 | 노드 크기/후광에 점수 매핑 필요 |
| 예외 처리 | 없음 | 경로 없을 때 메시지 표시 |
| 데이터 모델 | subject / core-concept 이분법 | Keyword / Data Node 이분 그래프 |
| 화살표(→) 의미 | 모호 | 슬롯 구분용(의미 없음)으로 명시 |
| 추천 UI 영역 | "📚 시스템 정보" 탭에 빈 통계 | 별도 영역 없음, 그래프에만 표현 |

## 5. 미결 사항

본 기능의 사용자 관점 동작은 위 규약으로 닫혔다. 다만 구현 단계에서는 다음이 후속 결정 필요:

- **경로 계산**: 프론트엔드에서 수행 ([MVP_SCOPE.md](MVP_SCOPE.md) §0). 알고리즘·hop 제한 등은 구현 시 결정.
- **GAT 점수**: 백엔드 모델 추론은 없음 ([MVP_SCOPE.md](MVP_SCOPE.md) §0). 시연용 mock 점수를 키워드별로 부여하는 방식을 결정해야 한다.
- "특히 추천도가 높은" 키워드를 **후광/뱃지로 강조하는 임계값** 정의.
- 키워드 **검색 UI**의 입력·매칭 규칙 (모든 키워드 선택 가능 전제).
- Data Node가 본 기능 외 다른 어디서 사용자에게 노출되는지 (별도 기능에서 다룰 수 있음).
