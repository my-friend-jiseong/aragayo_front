# 지식 그래프 구조 정의

본 문서는 CONREC 지식 그래프의 노드/간선 모델과 데이터 출처를 정의한다.
프론트엔드 시각화, 백엔드 모델(GAT 추천), 데이터 파이프라인 모두 이 정의를 따른다.

## 1. 노드 타입

그래프는 두 종류의 노드로 구성된다.

### 1.1 키워드 노드 (Keyword)
- **정의**: 학습 대상이 되는 개념 단위.
- **출처(최초 그래프 구성 시)**: 위키피디아 문서의 **타이틀**.
  - 예: `소프트웨어 공학`, `폭포수 모델`.
- **출처(이후)**: 사용자가 삽입한 **과목 정보**를 기준으로 새로 추출되어 추가됨.
  - 새로 생성된 과목 기반 키워드는 **재학습**을 통해 기존 데이터 노드들과 연결된다.
- **과목과의 관계**: 과목은 키워드의 한 종류이며, **별도 노드 타입이 아니다**.
  키워드의 출처(`source: 'wikipedia' | 'subject'`)로만 구분된다.
- **하이퍼링크**:
  - 위키 기반 키워드 → 해당 위키피디아 문서로의 하이퍼링크 보유.
  - 과목 정보 기반 키워드 → 원본 링크 없음. 하이퍼링크 생성 안 함.

### 1.2 데이터 노드 (Data Node)
- **정의**: 한 개 이상의 키워드를 포함하는 하나의 글(문서 단위).
- **출처(최초 그래프 구성 시)**: **블로그 글**.
- **하이퍼링크**: 원본 소스(블로그 글)로의 하이퍼링크 보유.

## 2. 간선 (Edge) 구조

- **방향성**: 모든 간선은 **단방향(Directed Edge)**.
- **연결 규칙**: 간선은 항상 **Data Node → Keyword** 방향으로만 존재한다.
  - 즉, "이 글(Data Node)이 이 개념(Keyword)을 다룬다"를 표현.
- **간접 연결**:
  - 키워드끼리는 직접 간선으로 연결되지 않는다.
  - 하나의 Data Node가 여러 Keyword를 참조하므로, **공유 Data Node를 통해 키워드들이 간접적으로 연결**된다.
  - 결과적으로 그래프는 **bipartite(이분 그래프)** 형태를 띤다.

## 2.1 시각화 시 적용 규칙

본 그래프 정의를 프론트엔드 시각화로 옮길 때 다음을 따른다.

- **두 노드 타입(Keyword, Data Node)을 모두 렌더링**한다. 어느 한쪽도 숨기지 않는다.
- 키워드 간 **직접 간선은 그리지 않는다**. 키워드끼리는 공유 Data Node를 통해서만 연결되는 것으로 표현한다.
- 간선은 **Data Node → Keyword** 방향의 화살표로 렌더링한다.
- 시각적 비중은 **키워드 > Data Node** (Keyword를 두드러지게, Data Node는 작고 흐리게).
- 시각화 단계의 강조는 본 구조 정의와 별개의 레이어이며, 어떤 강조도 구조를 거짓으로 표현해서는 안 된다.

## 3. 하이퍼링크 정책 요약

| 노드 타입 | 출처 | 하이퍼링크 |
|---|---|---|
| Keyword (위키 기반) | 위키피디아 문서 타이틀 | O — 위키피디아 문서 URL |
| Keyword (과목 정보 기반) | 사용자 입력 과목 | X — 링크 없음 |
| Data Node | 블로그 글 | O — 원본 블로그 URL |

## 4. 현재 프론트엔드 데이터 모델과의 갭

현재 [src/mockData.ts](../src/mockData.ts)는 `group: 'subject' | 'core-concept'` 이분법으로 구성되어 있어
본 문서의 `Keyword / Data Node` 모델과 일치하지 않는다. 향후 다음 작업이 필요:

- 노드 타입 필드를 `keyword` / `data-node`로 재정의.
- 키워드 출처 메타(`source: 'wikipedia' | 'subject'`) 추가.
- 간선 방향을 Data Node → Keyword로 정규화.
- 키워드 노드의 `wikiUrl`, 데이터 노드의 `blogUrl` 필드 정리.
