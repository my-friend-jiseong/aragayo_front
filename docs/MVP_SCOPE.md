# MVP 범위 정의

본 문서는 발표/시연 시점의 MVP 기능 범위를 정의한다.
구체 기능별 동작 명세는 각 기능 문서를 참고한다.

## 0. 프로젝트 컨텍스트 (전제 조건)

- **프로젝트 성격**: 해커톤 출품용 시연 프로젝트.
- **백엔드 연동 없음**: 본 프로젝트는 **프론트엔드 시연에 중점**을 둔다.
  실제 GAT 모델 추론, 저장소 영속화, 외부 API 호출은 수행하지 않는다.
- **모든 데이터는 mock**: 그래프 노드/간선, GAT 추천 점수, 사용자 상태 등 모든 데이터는
  프론트엔드 내부 mock으로 시연한다.
- **상태 저장**: 사용자 마킹·커리큘럼 저장 등 영속이 필요한 데이터는
  브라우저 로컬(예: `localStorage`)에 저장한다.
- **경로 계산**: A↔B 사이 경로 탐색 로직은 프론트엔드에서 수행한다.

이 컨텍스트는 모든 기능 명세의 상위 제약이다. 기능 설계 시 백엔드 호출이 필요해 보이는
지점은 모두 mock/로컬 대안으로 치환해야 한다.

## 1. MVP 기능 (우선순위 순)

### 1-1. (P1) 메인 화면 — 그래프 화면
지식 그래프 기반 시각화 화면. [FEATURE_PATH_VISUALIZATION.md](FEATURE_PATH_VISUALIZATION.md) 명세를 따른다.
- 두 키워드 선택 → 사이 경로 시각화 + GAT 추천 통합 (Q1+Q2 결과).
- 그래프 구조: [GRAPH_ARCHITECTURE.md](GRAPH_ARCHITECTURE.md) 기반.

(확인 필요) 두 키워드 경로 시각화가 본 화면의 일부인지, 별도 진입점인지 명확화 필요.

### 1-2. (P2) 학습 완료된 / 학습 예정인 노드 표시
사용자가 그래프상 키워드 노드에 **"학습 완료"** 또는 **"학습 예정"** 상태를 마킹할 수 있고,
시각적으로 구분되어 표시된다.
- 상세 명세는 후속 작성 예정 (`FEATURE_NODE_STATUS.md`).

### 1-3. (P3) 내 커리큘럼 (경로 저장)
사용자가 선택한 두 키워드 사이 경로를 **"내 커리큘럼"**으로 저장하고 다시 불러올 수 있다.
- 상세 명세는 후속 작성 예정 (`FEATURE_MY_CURRICULUM.md`).

## 2. MVP에 포함되지 않는 기능 (Non-MVP)

### 2-1. 사용자 "현재 이해 상태" 세분화 입력 (구 Q3)
"안다 / 모른다 / 진행 중" 같은 세밀한 이해 상태를 키워드별로 입력받는 기능은 MVP 외이다.
GAT 모델 입력 신호로 쓸 가능성이 있으나, MVP 단계에서는 §1-2의 단순 "완료/예정" 마킹으로 대체된다.

### 2-2. 학습 자료(블로그/영상) 추천 별도 영역
Data Node를 사용자에게 노출하는 별도 UI(추천 자료 리스트 등)는 MVP 외이다.
GAT 추천은 §1-1의 그래프 위 키워드 시각 표현으로만 전달된다.

## 3. 후속 결정 필요 사항
- (P1) 두 키워드 경로 기능의 화면상 위치 — 메인 화면 내장 vs. 별도 진입점.
- (P1) 메인 화면의 초기 상태(아무것도 선택 안 했을 때) 표시 내용.
- (P2) 상태 종류·시각 표현·마킹 방식.
- (P3) 저장 단위·UI 위치·불러오기 방식·다중 저장 여부.

(구 Q4) 백엔드 연동은 §0에 따라 본 프로젝트 범위에서 제외됨.
